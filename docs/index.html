模块和网站分别做什么

            
主模块：装进 Shadowrocket 后才会生效。它负责接管 script.hub、grandpaniu.script.hub，并在支持的网站上识别模块链接。

            
工具网站：不替代主模块。它适合新手找入口、复制链接、批量扫描文本，也适合在模块没装好时先生成 Shadowrocket 安装链接。

            
网页跳转识别：网站可以解析你粘贴的跳转链接；主模块启用后，可以在支持的网站页面内自动发现隐藏的模块链接并显示助手面板。

            
支持段落

            
General、Header Rewrite、URL Rewrite、Body Rewrite、Script、MITM 都会被内容分析统计。真正的跨工具转换交给 Script-Hub 官方脚本地址处理，本项目不内嵌官方源码。

            
Pages 还 404 怎么办

            
仓库需要在 GitHub 设置里把 Pages 来源设为 Deploy from a branch，分支选择 gh-pages，目录选择 / (root)。最新工作流会把静态网站发布到这个分支。

          

        

      


      

        

          
小白路径

          

            
1. 安装主模块点击绿色按钮，Shadowrocket 弹窗后确认导入。

            
2. 启用模块和 MITMShadowrocket 模块列表启用，证书也要信任。

            
3. 选择模块导入从模块库点“转换后导入”，或粘贴自己的链接。
